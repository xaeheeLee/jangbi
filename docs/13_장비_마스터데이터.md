## 1. 분류 체계

장비는 **카테고리(3종) + 모델코드**의 2단 구조입니다.

- **카테고리**가 1차 분류(미니 / 트랙 / 타이어)이며, 서로 다른 카테고리는 **매칭되지 않습니다**.
- **모델코드**는 카테고리 내부의 규격(톤급) 순서이며, `sort_order`가 클수록 큰 장비입니다.

## 2. 매칭 규칙 (카테고리 + 모델 단위, "이상")

발주의 요구 조건은 `required_category`(필수) + `required_model`(선택, **최소 모델**)로 표현합니다. **여러 허용 장비를 등록할 수 있으며(`job_equipment_options`), 이 경우 옵션 중 하나라도 만족하면 매칭(OR)** 됩니다(§2.1).

- **카테고리**: 발주 카테고리와 **동일**해야 매칭(미니 ↔ 트랙 ↔ 타이어 상호 불가).
- **모델**: `required_model`이 지정되면, 기사 보유 모델의 `sort_order`가 **요구 모델의 `sort_order` 이상(≥)**이어야 매칭.
  `required_model`이 `NULL`이면 해당 카테고리 **전체**가 매칭 대상.

```
매칭 가능 ⇔
  driver.equipment_category = job.required_category
  AND ( job.required_model IS NULL
        OR rank(driver.equipment_category, driver.equipment_model)
           >= rank(job.required_category, job.required_model) )
```

> 예) 발주 `굴삭기(트랙) 06LC 이상` → `track` 카테고리에서 `sort_order ≥ 4`(06LC·08LC·10LC·380LC … 730LC) 보유자만 매칭.

> **정책 메모:** 현재 규칙은 "요구 모델 = 최소 규격(이상)" 해석입니다. 추후 "정확히 일치" 또는 "상한 지정"이 필요하면 `sort_order` 비교 연산자만 바꾸면 됩니다(데이터 구조 변경 불필요).

### 2.1 다중 허용 (OR 매칭)

발주자가 장비를 가리지 않을 때, 한 일감에 **서로 다른 카테고리/모델을 여러 개** 등록할 수 있습니다(예: `미니 035` · `트랙 02` · `트랙 03`). 옵션은 `job_equipment_options`(대표 `required_*` + 추가 옵션)에 저장됩니다.

```
매칭 가능 ⇔ 기사 보유 장비가 옵션 집합 중 "하나라도" 만족
  ∃ opt ∈ {required_* + job_equipment_options} :
      driver.equipment_category = opt.category
      AND ( opt.min_model IS NULL
            OR rank(driver.equipment_category, driver.equipment_model)
               >= rank(opt.category, opt.min_model) )
```

> 옵션 1개면 단일 지정(§2)과 동일하게 동작합니다. "아무거나 1대만 와도 OK"인 발주에 사용.

  ('tire',  '굴삭기(타이어)',  3);

  ('mini','020','020',4),('mini','025','025',5),('mini','030','030',6),
