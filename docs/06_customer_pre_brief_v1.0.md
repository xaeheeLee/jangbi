# 장비나라 앱 — 2차 미팅 사전 검토 자료

> **문서 목적**
> 2차 미팅에서 다룰 핵심 안건을 미리 검토해 주시기 위한 압축본입니다.
> 함께 결정해야 할 항목 위주로 정리했으니, 미팅 전에 한 번 훑어주시면 감사드립니다.

---

## 1. 미팅 목적 (요약)

1차 미팅 이후 개발측에서 검토 가능한 항목은 **미리 확정 진행**으로 두었고, 고객 결정이 필요한 항목만 미팅에서 다루려고 합니다. 추가로 손든 사람 기능 / 작업일지·정산 모듈 포함 여부에 따라 견적 시나리오를 4가지로 준비했습니다.

---

## 2. 1차 미팅 합의사항 — 재확인만

미팅에서는 짧게 재확인하고 넘어갈 항목입니다.

| 항목 | 합의 내용 |
|------|----------|
| 장비 종류 | 1차는 포크레인만, DB는 확장 가능 구조 |
| 회원 가입 | 면허/자격증/장비 보유증 업로드 → 담당자 승인 |
| 충전금/결제 | 1차 제외 (세무사 논의 후 결정) |
| 회원 유형 | 1차는 기사만 (업체 회원 제외) |
| 관리자 작업 | 별도 웹 없이 **앱 내 관리자 모드**로 구현 |




---



## 3. 개발측 확정 진행 (별도 이의 없으시면 그대로 진행)

### 발주 화면
작업 시간 범위 + 사용시간 칩 / 작업 종류 태그(다중 선택) / 이용 금액 입력 / 임시저장·사진 첨부·3단계 스텝퍼

### 매칭 흐름
캘린더 4단계 색상(요청·진행·완료·휴무) / 완료된 일감 흐림 처리 / 폴더블폰 대응

### 회원가입
약관 4종 / 자격증·보유증 업로드 + 관리자 승인 / 가입 승인 대기 중에는 대기 화면만 표시 (모든 기능 차단) / **아이핀 인증 1차 제외** (휴대폰 인증만)

### 우선배차권
일감 등록 시 등록자에게 1장 발급 / 발급일로부터 30일 만료

### 1차 출시에서 빠지는 항목
- **커뮤니티 채팅** — 1차 제외, 향후에도 미정
- **작업일지 / 월별 정산 / 거래명세서** — 1차 제외, 2차 별도 모듈로 견적 제시
- **신고 시스템** — 제외
- **PC/태블릿 관리자 대시보드** — 1차 제외 (앱 내 관리자 모드로 대체)
- **충전금 / 인앱 결제** — 1차 제외 (세무사 논의 후 결정)

## 4. 함께 결정해야 할 항목

미팅 전에 각 항목에 대한 입장을 정리해 오시면 좋습니다. 항목별로 **선택지**와 **개발측 의견**을 함께 적어두었습니다.

### 🟦 우선순위 1 — 매칭 모델

#### Q1. 손든 사람 기능 도입 여부 (★ 핵심)
발주자 화면에 손든 기사 리스트가 실시간으로 떠서 직접 채택할 수 있는 기능.
- **A. 제외** — 자동 매칭만 (공정성 우선, 기존 합의안)
- **B. 포함** — 발주자 주관 채택 가능 (+3주 견적)

> **개발측 의견:** 우선배차권 보유자가 매칭권을 받아도 발주자가 무시하고 손든 기사를 채택하면 배차권 가치가 소멸 → 환불 정책 필요. "공정 배차"가 앱의 핵심 가치라면 A안 권장. 그래도 발주자 선택 폭이 중요하다면 B안.

#### Q2. 매칭 점수 공식
- 기존 합의안: 별점 70% + 거리 30%
- 고객 예시안: 콜 수행 40% + 정시 배차 30% + 회원 평가 50% - 취소 패널티 20% (거리 미반영)

> **결정 필요:** 거리 반영 여부 / 4지표 도입 시 정확한 비율

#### Q3. 우선배차 시간 — 푸시 알림 지연 안내 (참고)

1차 미팅에서 **30초로 합의된 항목**이며, 별도 변경이 없는 한 30초로 진행됩니다. 다만 운영 단계에서 푸시 알림이 디바이스 상태에 따라 지연될 수 있어, 사전에 도달 시간 범위를 공유드립니다.

**디바이스별 푸시 도달 시간 (백그라운드/종료 상태 기준)**

| 상태 | 최소 지연 | 최대 지연 | 30초 중 가용 시간 |
|------|---------|---------|---------------|
| iPhone (정상) | 1초 | 3초 | 약 25~27초 ✅ |
| 갤럭시 (정상) | 3초 | 10초 | 약 18~22초 ⚠️ |
| 갤럭시 (절전모드/디바이스 케어) | 10초 | 30초+ | 0~15초 ❌ |
| 앱 종료 상태 | 위 시간 + 2~3초 | 위 시간 + 2~3초 | 더 짧음 |

> **참고 사항**
> - 앱이 켜져 있는 경우 Realtime 구독으로 0.1~0.3초 내 즉시 표시 (푸시 무관)
> - 한국 기사층의 갤럭시 비중이 높고 절전모드가 기본 활성화되어 있어, 30초 우선배차는 **앱을 열어두고 대기 중인 활동성 높은 사용자 간 경쟁** 구도가 될 가능성이 큽니다.
> - 보완책: 고우선순위 FCM 사용 / 앱 첫 실행 시 절전모드 예외 등록 안내 / 카카오 알림톡 병행 발송(별도 비용) 등 검토 가능

### 🟦 우선순위 2 — 회원 정보 및 가입 정책

#### Q4. 회원가입 시 받을 정보 항목

이름 / 휴대폰 / 비밀번호 / 약관 / 면허·자격증 사진 / 장비 보유증 / 장비 종류·크기 — **확정**

아래 항목은 받을지/안 받을지 결정 필요:

| # | 항목 | 개발측 의견 |
|---|------|----------|
| 1 | 생년월일 | 받음 권장 (연령 통계, 본인 확인) |
| 2 | 성별 | 선택 사항 |
| 3 | 이메일 | 선택 권장 (알림·계정 복구용) |
| 4 | 자격증 종류 명시 | 명시 권장 (중장비조종사면허 등) |
| 5 | 사업자 정보 | 추후 세금계산서 발행 대비 받음 권장 |
| 6 | 활동 지역 | 받음 권장 (매칭 정확도) |
| 7 | 차량번호 | 선택 |
| 8 | 계좌번호 | 매칭 후 공개 권장 (가입 시 X) |
| 9 | 경력 사항 | 선택 |

#### Q6. 정회원 등급 기준
- 가입 시 자동 정회원? (자격증 승인 = 정회원)
- 별도 신청 + 관리자 추가 승인?
- 활동지수 N점 이상 자동 승급?

> **결정 필요:** 정회원과 일반회원의 차등 기능 정의

#### Q7. 활동지수 산정 기준
일감 등록 / 손들기 / 매칭 성공 / 작업 완료 / 취소 페널티 등에 몇 점씩 부여할지.

> **결정 필요:** 점수 부여 정책, 최대치, 감소 정책

### 🟦 우선순위 3 — 발주 화면 세부

#### Q9. 이용 금액 입력 모델
- **A. 사전 가격 제시** — 발주자가 정한 가격 보고 기사가 지원/거부
- **B. 견적 요청** — 발주자 희망가 표시, 기사가 다른 가격 제시 가능
- **C. 혼합** — 우선배차는 정가, 일반 매칭은 협상

#### Q10. 작업 종류 태그 목록
현재안: 가위 / 고무궤도 / 그라플 / 다짐기 / 돌집게 / 뿌레카 / 지게발 / 코아(천공) / 기타
> **결정 필요:** 추가·삭제 항목



---



### 🟦 우선순위 4 — 향후 확장

#### Q11. 작업일지 + 정산 + 거래명세서 추가 시점
사업 전략상 핵심 락인 도구로 보임. 추가 약 5주, 별도 비용.
- **A. 1차 포함** — 전체 일정 길어짐
- **B. 1차 출시 후 즉시 2차 개발 진행** (개발측 권장안)
- **C. 운영 안정화 후 추후 결정**

#### Q12. 평점/평가 시스템 세부 정책
평가 시점 / 평가 항목(별점만 vs 별점+코멘트) / 미작성 시 처리 / 양방향 평가 여부
