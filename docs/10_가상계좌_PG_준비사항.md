# 가상계좌 / 포인트 충전 — PG 계약·심사 준비사항 (고객 전달용)

> **목적:** 전중배 앱의 가상계좌 충전(포인트 적립)을 위해 PG사(토스페이먼츠 권장) 계약·심사 시
> 고객(사업자)이 준비해야 할 사항을 정리한 문서입니다.
> **개발자는 API 연동(기술)만 담당하며, 계약·심사·정산 주체는 사업자(고객)입니다.**

---

## 1. 왜 고객(사업자)이 계약 주체인가

- PG 정산금(충전된 금액)은 **사업자 명의 계좌로 입금**됩니다.
- 심사 시 사업자등록증·대표자 정보·정산계좌가 필요합니다.
- 따라서 **계약 명의자 = 전중배 운영 사업자**이며, 개발자는 발급받은 키로 연동만 진행합니다.

---

## 2. 계약·심사에 필요한 서류 (토스페이먼츠 기준)

| 구분 | 서류 / 정보 | 비고 |
|------|------------|------|
| 사업자 확인 | **사업자등록증** | 개인사업자 / 법인 모두 가능 |
| 대표자 확인 | **대표자 신분증 사본** | |
| 정산 계좌 | **사업자 명의 통장 사본** | 충전금이 입금될 계좌 |
| 영업 정보 | 업종, 제공 서비스 설명, 앱/웹 정보 | "중장비 배차 중개 + 포인트 충전" |
| 온라인 판매 | **통신판매업 신고증** | 포인트(전자상거래성) 판매 시 요구될 가능성 높음 |
| 정산 조건 | 희망 정산주기 | 가상계좌는 보통 입금확인 후 D+1~D+2 |

> 심사 기간: 보통 **영업일 3~5일**. 영업 실태(서비스 내용·정산 방식)를 함께 봅니다.

---

## 3. 수수료 (대략)

| 항목 | 수수료 (참고치) |
|------|----------------|
| 가상계좌 발급/입금 | **건당 약 300원 + VAT** (PG사 간 큰 차이 없음, 협의 가능) |
| (카드결제 병행 시) | 약 2.x% + VAT |

> 충전 시 사용자에게 부과하는 **부가세 10%**(예: 3만원 충전 시 3.3만원 입금)와 PG 수수료는 별개입니다.
> PG 수수료는 사업자 정산 시 차감되는 비용입니다.

---

## 4. ⚠️ 반드시 확인이 필요한 법적 사항 (선불충전금)

포인트를 **미리 충전해 보관**하는 구조는 **선불전자지급수단(선불충전금)** 에 해당할 수 있습니다.

- **전자금융거래법** 적용 검토 필요 — 발행 잔액/총 발행액이 일정 규모를 넘으면 **금융위원회 등록** 의무가 생길 수 있습니다.
- 1인 운영 초기 소규모라면 **등록 면제 대상**일 가능성이 높으나, **법무/세무 전문가 확인을 권장**합니다.
- **특히 중요한 변수:** 포인트(소개비로 받은 것 포함)를 **현금으로 출금(환급)** 가능하게 할 것인지.
  - 앱 내 사용(구독료·소개비 지불)만 허용 → 규제 부담 ↓
  - 현금 출금 허용 → 규제·정산 부담 ↑ (이 경우 별도 정산 설계 필요)
  - → **이 항목은 고객 결정 필요 (질문지 Q11 참조)**
