## 1. 제품 개요

### 1.1 정의
포크레인(굴착기) 기사 간 **일감 등록·배차 매칭** 모바일 플랫폼. 단일 사용자 유형(중장비 기사)이 일감을 올리고(발주) 동시에 배차를 받을 수 있습니다.

### 1.2 비즈니스 모델 (전중배 = 기사 간 배차 중개)
- 발주자 A가 직접 수행 못 하는 일감을 등록 → 기사 B가 배차받아 수행
- B는 일감 금액의 **10%를 소개비로 A에게 포인트 지급** (플랫폼 수취 0원 [확인필요 Q2])
- 일당 본금(예: 50만원)은 앱 외부에서 직접 수령 (앱은 "계산서 발행 권장" 안내만)

### 1.3 수익 구조

| 구분 | 내용 | 귀속 |
|------|------|------|
| 구독료 | **하루 100p 차감** (정회원 유지 조건) | 플랫폼 |
| 충전 부가세 | 충전액의 **10%** (3만원 충전 시 3.3만원 입금 → 3만p 발급) | 플랫폼/세무 |
| 일감 소개비 | 일감 금액의 **10%** (B→A) | 발주자 A (플랫폼 0원) |

### 1.4 기술 스택

| 구분 | 기술 |
|------|------|
| 모바일 | Flutter (iOS / Android) |
| 상태관리 | Riverpod / 라우팅 go_router |
| 백엔드·DB | Supabase (PostgreSQL + Realtime + Auth + Storage) |
| 결제 | 토스페이먼츠 가상계좌 (포인트 충전) |
| 지도 | 카카오맵 (일감 위치 표시용) |
| 푸시 | FCM |
| 스케줄러 | pg_cron |
| 관리자 | **PC 웹 페이지 (별도 산출물)** |

> 거리 기반 매칭 폐지로 매칭에는 GPS를 사용하지 않으며, 지도는 일감 위치 표시 용도로만 사용합니다.

## 3. 모바일 앱 기능 명세

### 3.1 인증 / 회원가입

| 기능 | 상세 |
|------|------|
| 로그인 | **핸드폰번호(ID) + 비밀번호**. 세션 장기 유지(자동 풀림 없음), 로그아웃 시 종료 |
| 본인 인증 | **카카오 알림톡 OTP** [추후 구현 — 1차는 보류, 인프라 준비 후 적용] |
| 회원가입 단계 | ① 약관 동의 → ② 정보 입력 → ③ 서류 업로드 → ④ 신청 완료 → ⑤ 관리자 승인 대기 |
| 가입 정보 | 이름, 핸드폰번호, 비밀번호, 자격증 종류, 사업자 정보, **사업자 이메일·사업자 계좌번호** |
| 업로드 서류(5종) | ① 건설기계 조종사 면허증 ② 건설기계 등록증 ③ 보험가입 증명서 ④ 사업자 등록증 ⑤ 본인 사진(어깨 보이는 상반신, 3개월 내) |
| 승인 흐름 | 관리자(PC) 승인 전까지 전체 차단 화면. 승인 시 정회원 활성 |

> 가입 정보·서류는 고객이 추후 추가/변경 요청 가능(변경관리 절차 적용).

### 3.2 프로필

| 기능 | 상세 |
|------|------|
| 프로필 조회/수정 | 이름, 장비 정보(궤도/바퀴, 소/중/대형), 연락처, 계좌(매칭 후 공개) |
| 별점 표시 | 유효별점(아래 3.6) + "신규" 뱃지(첫 5회) |
| 활동 통계 | 수행 일감 수, 보유 포인트, 보유 배차권 등 |

### 3.3 일감 등록 (발주)

| 기능 | 상세 |
|------|------|
| 입력 항목 | 작업 일시, 지역명/주소, 작업 종류(다중 태그), 장비 조건(궤도형/바퀴형·크기), 일감 금액, 메모, 사진 첨부 |
| 작업 종류 태그 | 가위/고무궤도/그라플/다짐기/돌집게/뿌레카/지게발/코아(천공)/기타 |
| 위치 | 카카오맵 핀 + 주소 검색(Geocoding) — 표시용 |
| 등록 효과 | **우선 배차권 1장 자동 발급(발급 후 30일 만료)** + 조건 맞는 보유자 존재 시 우선배차 윈도우 개시 |
| 발주 제한 | 일감 올리기는 무제한 [확인필요 Q8] |
| 결제 방식 표기 | 직수(직접수령)/싸인지/직접청구/현금결제 중 1개 표시 + "계산서 발행 권장" 안내 |

### 3.4 일감 목록 / 상세

| 기능 | 상세 |
|------|------|
| 목록 | 필터(장비 조건), 정렬(날짜순), 실시간 갱신(Realtime) |
| 배차 완료 표시 | "배차완료" 배지 + 블러 처리 |
| 상세 | 위치 지도, 작업 정보, 장비 조건, 금액, 결제방식, 일정 충돌 경고 |
| 지원 버튼 | 우선 지원(배차권 보유 시) / 일반 지원 분기. 포인트 부족 시 비활성 + 충전 안내 |

### 3.5 매칭 (핵심 로직)

**우선순위 (높은 순):**
1. **프리미엄 배차권** (관리자 부여, 고객에게 비공개)
2. **우선 배차권 + 별점**
3. **일반 배차 (선착순)**

| 항목 | 상세 |
|------|------|
| 우선배차 윈도우 | 일감 등록 시 조건 맞는 우선배차권 보유자가 있으면 **30초** 윈도우 개시 |
| 우선 지원 | 30초 내 우선배차권 보유자만 지원 가능 |
| 채택 | 30초 종료 후 **별점(유효별점) 최고**인 지원자 채택. 동점 시 **선착순** |
| 프리미엄 처리 | 프리미엄 배차권 보유자가 지원 시 별점과 무관하게 **최우선 채택**. 단 **즉시 채택하지 않고 30초 종료 후 확정**(비공개 유지). 프리미엄 2명 이상이면 **먼저 지원한 사람** |
| 일반 배차 | 우선배차권자 미존재 시 또는 윈도우 종료 후 일반 공개 → **선착순 즉시 매칭** [확인필요 Q8: 30초 내 우선지원자 없으면 일반 전환] |
| 1일 1건 | 기사는 **하루 1건만 배차** 가능(일정 충돌 체크) [확인필요 Q8] |
| 탈락자 | 우선배차권 반환(만료일 최소 보장) |

> 프리미엄 배차권 존재·작동 방식은 일반 사용자에게 노출되지 않도록 설계(점수 비공개, 즉시 채택 금지).

### 3.6 별점 / 평가

| 항목 | 상세 |
|------|------|
| 평가 주체 | 일감 수행 후 **발주자가 0~5점** 부여 |
| 매칭 점수 | **별점 100%** (거리 반영 폐지) |
| 콜드스타트 | **베이지안 가중평균** — 유효별점 = (평가수×본인평균 + m×전체평균C)/(평가수+m), 초기 C=3.5·m=5(설정값) |
| 신규 뱃지 | 첫 5회 일감 동안 "신규" 표시 |
| 특별 별점 | 관리자가 사유와 함께 **+1 / +2 가산점** 부여(유효별점 위에 가산) |

### 3.7 포인트 / 충전 / 소개비

| 기능 | 상세 |
|------|------|
| 단위 | **1 포인트 = 1원** |
| 충전 | 토스페이먼츠 **가상계좌** 입금 확인 시 포인트 발급. 충전액 + **부가세 10%** 입금(예: 3만원 충전 → 3.3만원 입금 → 3만p 발급) |
| 입금 확인 | 가상계좌 입금통보(webhook) 자동 처리 |
| 구독 차감 | **하루 100p 자동 차감**, 잔액 0 시 정회원 박탈 [확인필요 Q1] |
| 소개비 | 배차 확정 시 일감 금액의 **10%를 B→A로 포인트 이전** + 배차권 사용은 **동시 처리** [확인필요 Q2·Q3] |
| 부족 시 | 소개비 낼 포인트 부족하면 지원 불가(버튼 비활성 + 충전 안내) |
| 내역 | 충전/차감/소개비 입출 내역 조회 |

### 3.8 캘린더 / 일정

| 기능 | 상세 |
|------|------|
| 월간 달력 | 배차받은 일정 / 내가 올린 일감 색상 구분 표시 |
| 일별 상세 | 해당일 일정 목록 + 상세 이동 |
| 충돌 경고 | 같은 날 2건 이상 시 경고. 지원 시 충돌 다이얼로그 → 강제 진행 옵션 |

### 3.9 푸시 알림 (FCM)

| 이벤트 | 대상 |
|--------|------|
| 새 일감 등록 | 조건 맞는 기사(우선권 보유자 우선) |
| 매칭 성공/실패 | 채택 기사 + 발주자 / 탈락 기사 |
| 일반 공개 전환 | 조건 맞는 기사 |
| 배차권 만료 임박 | 만료 임박 보유자 |
| 포인트/정회원 경고 | 잔액 임박·박탈 대상 |

## 4. PC 관리자 웹 페이지 (별도 산출물 · +3주)

## 5. 범위 제외 (Out of Scope) ⭐

본 계약에 **포함되지 않는** 항목입니다. 필요 시 별도 견적/계약으로 진행합니다.

- "손든 사람" 기능 (1차 미구현 확정)
- 작업일지 / 정산 / 거래명세서 모듈
- 커뮤니티 채팅 (그룹/지역방)
- 포인트 **현금 출금(환급)** 기능 [확인필요 Q11 — 허용 시 별도 정산 설계]
- 카카오 알림톡 OTP 인증 (1차 보류, 추후 구현)
- 신고 시스템
- 웹/태블릿용 **사용자(기사) 화면** (관리자 웹은 별도 포함)
- 앱 외부 결제/세금계산서 자동 발행 (앱은 "계산서 발행 권장" 안내만)
- 기사/업체 회원 분리, 아이핀 인증
