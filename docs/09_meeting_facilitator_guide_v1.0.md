## ② 결정 항목 Q1 ~ Q12

> 형식: **질문 → 선택지 → 🟢 개발측 권장 → ↪︎ 다른 방향 시 대응 → ✍️ 결정란**

### 🟦 우선순위 1 — 매칭 모델 (개발 방향 좌우, 시간 우선)

#### Q1. 손든 사람 기능 도입 여부 ★최우선
- A. 제외 (자동 매칭만, 기존 합의) / B. 포함 (발주자 직접 채택, **+3주**)
- 🟢 **권장: A안.** 우선배차권 받은 기사가 손든 기사에 밀리면 배차권 가치 소멸 → 환불 정책 필요. "공정 배차"가 핵심 가치면 A.
- ↪︎ 고객이 B 강력 희망 시: **시나리오 B/D로 견적 +3주 명시**, 우선배차권 환불 정책 같이 설계해야 함을 못박기.
- ✍️ 결정: ____________________

#### Q2. 매칭 점수 공식 ★큰 건
- 기존 합의: 별점 70% + 거리 30% / 고객 예시안: 콜수행40%+정시배차30%+회원평가50%-취소20% (거리 미반영)
- 🟢 **권장: 별점 + 거리 유지.** 단 먼저 물을 것 → **"포크레인 기사가 같은 시/도 안에서 도는가, 광역 이동인가?"** 광역이면 거리 비중 낮추거나 뺀다.
- ↪︎ 고객이 4지표안 원하면: 지표는 받되 **비율은 개발측이 초안 제시 → v2.2 확정**으로. 현장에서 숫자 확정하려 하지 말 것.
- ✍️ 결정: 활동 반경 = ________ / 공식 방향 = ____________________

#### Q3. 우선배차 시간 (참고 — 30초 합의됨)
- 변경 없으면 30초 진행. **결정 강요 X**, 푸시 지연 사실만 공유.
- 🟢 **권장: C안 (운영자 설정값, 출시 후 데이터 보고 조정).** 보완책 3종 안내: 고우선순위 FCM / 첫 실행 시 절전모드 예외 안내 / 카카오 알림톡 병행(별도 비용).
- 📊 푸시 도달 (백그라운드 기준): iPhone 1~3초(25~27초 가용) / 갤럭시 정상 3~10초(18~22초) / **갤럭시 절전 10~30초+(0~15초 ❌)**. 앱 켜진 상태는 Realtime 0.1~0.3초 즉시.
- ✍️ 결정: ________ 초 / 알림톡 검토 ☐

### 🟦 우선순위 2 — 회원 정보 및 가입 정책

#### Q4. 회원가입 시 받을 정보
- 확정(자동): 이름·휴대폰·비번·약관·자격증사진·보유증·장비종류/크기
- 🟢 **권장 매핑:** 생년월일 받음 / 성별 선택 / 이메일 선택 / 자격증종류 명시(중장비조종사면허 등) / 사업자정보 받음(세금계산서 대비) / 활동지역 받음(매칭정확도) / 차량번호 선택 / 계좌 매칭 후 공개 / 경력 선택
- ↪︎ 고객이 "최소만" 원하면: 생년월일·활동지역·사업자만 사수, 나머지 양보 가능.
- ✍️ 결정: (체크시트에 표시받기)

#### Q6. 정회원 등급 기준
- 🟢 **권장: 가입(자격증 승인) = 자동 정회원으로 단순화.** 활동지수 기반 승급은 2차로 미룸.
- ↪︎ 고객이 등급 차등 원하면: **정회원 vs 일반회원 "기능 차이"가 뭔지** 구체적으로 받아오기 (예: 우선배차권 사용 권한). 차이 정의 없으면 등급제 자체가 무의미.
- ✍️ 결정: ____________________

#### Q7. 활동지수 산정 기준
- 🟢 **권장: 단순 가점제로 시작 + 운영 데이터 보고 조정.** 개발측 초안 제시 → 합의.
  - 초안 예: 일감등록 +5 / 매칭성공 +10 / 작업완료 +15 / 손들기 +2 / 취소 -20 / 상한 100
- ↪︎ 현장에서 숫자 못 정하면 "초안대로 시작, 운영 중 조정"으로 합의.
- ✍️ 결정: ____________________

#### Q9. 이용 금액 입력 모델

## ③ 견적·일정 시나리오 (15분, 택1 유도)

| 시나리오 | 손든(Q1) | 정산모듈(Q11) | 기간 |
|---------|:---:|:---:|:---:|
| **A. 표준 (권장)** | 제외 | 2차 별도 | **24주 (6개월)** |
| B. 손든 추가 | 포함 | 2차 별도 | 27주 |
| C. 정산 포함 | 제외 | 포함 | 29주 |
| D. 풀스코프 | 포함 | 포함 | 32주 |

> **권장 흐름:** A안 1차 출시 → 4~6주 운영 안정화 → 2차 모듈 추가 (총 35주, 8~9개월)
> **변동 범위:** 공격적 20주 / 표준 24주 / 여유 28주 — **± 4주 가능성 미리 안내**

**마일스톤(A안):** 4주 회원가입 → 8주 일감등록 → 13주 매칭 → 16주 푸시 → 20주 캘린더 → **24주 1차 출시**

**운영비(월):** 초기 ~300명 3.7~4만원 / 1,000명 4~4.5만원 / 3,000명 4.5~5만원
**1회성:** Apple 연 12.9만원 / Google Play 1회 3.5만원

**운영비 구성 (비용 질문 시 즉답 카드)**

| 항목 | 비용 | 메모 |
|------|------|------|
| Supabase Pro | 약 3.5만원 | 고정비 (8GB DB, 100K MAU, 자동백업) |
| SMS 인증 | 0.4~1.2만원 | 건당 ~20원, 월 200~600건 가정 |
| 카카오맵 JS·로컬 API | **0원** | 지도 무제한 / 주소변환 일 10만건 무료 |
| Firebase FCM | **0원** | 무제한 무료 |

→ **사실상 고정비 = Supabase + 문자뿐.** 지도·푸시 무료.

**카카오 로컬 API 호출량 (고객 일감 빈도 반영 — "비싸지 않냐" 질문 대비)**

| 시즌 | 평일(22일) | 주말(8일) | 월 호출량 |
|------|:---:|:---:|:---:|
| 비수기 (9개월) | 30건/일 | 5건/일 | 약 700건/월 |
| 겨울 (3개월) | 5건/일 | 5건/일 | 약 150건/월 |

→ 피크 월도 무료 한도(일 10만건=월 ~300만건)의 **0.02%**. **3,000명까지 무료 한도 내 안전.** 단 카카오 로고/저작권 표기 유지 필수.
