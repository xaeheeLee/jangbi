## 미팅에서 결정할 것 — 핵심 5가지

### ① 손든 사람 기능, 넣을까요?
발주자가 손든 기사 리스트를 보고 **직접 채택**할 수 있는 기능입니다.
- ☐ A. **제외** (자동 매칭만, 기존 합의안) — 공정성 우선, 일정 변동 없음
- ☐ B. **포함** — 발주자 재량 ↑, 단 **+3주 추가** · 우선배차권 환불 정책 필요

> 개발측 의견: 우선배차권을 받은 기사가 손든 기사에 밀리면 배차권 가치가 사라집니다. "공정 배차"가 핵심 가치라면 A안 추천.

### ② 작업일지·정산·거래명세서, 언제 넣을까요?
사업 락인 도구로 보이지만 1차에 넣으면 일정이 길어집니다.
- ☐ A. **1차에 포함** (총 29주)
- ☐ B. **출시 후 바로 2차 개발** (개발측 권장)
- ☐ C. **운영 안정화 후 추후 결정**

### ③ 회원가입 시 받을 정보 (체크해 주세요)
이름·휴대폰·비밀번호·약관·자격증·장비 정보는 **이미 확정**입니다. 아래만 결정 필요:

| 항목 | 받을지 |
|---|---|
| 생년월일 | ☐ 받음 / ☐ 안 받음 |
| 성별 | ☐ 받음 / ☐ 안 받음 |
| 이메일 | ☐ 필수 / ☐ 선택 / ☐ 안 받음 |
| 사업자 정보 (세금계산서용) | ☐ 받음 / ☐ 안 받음 |
| 활동 지역 | ☐ 받음 / ☐ 안 받음 |
| 차량번호 | ☐ 받음 / ☐ 안 받음 |
| 경력 사항 | ☐ 받음 / ☐ 안 받음 |

### ④ 작업 종류 태그, 이대로 괜찮을까요?
현재안: **가위 / 고무궤도 / 그라플 / 다짐기 / 돌집게 / 뿌레카 / 지게발 / 코아(천공) / 기타**
- ☐ 이대로 OK
- ☐ 추가/삭제 필요 (목록: ____________)

### ⑤ 이용 금액, 어떤 방식으로 받을까요?
- ☐ A. **사전 가격 제시** — 발주자가 가격을 정함, 기사는 지원/거부
- ☐ B. **견적 요청** — 발주자 희망가 표시, 기사가 다른 가격 제시 가능
- ☐ C. **혼합** — 우선배차는 정가, 일반 매칭은 협상

## 견적 (4가지 시나리오)

| 시나리오 | 손든 사람 | 작업일지·정산 | 개발 기간 |
|---------|---------|------------|--------|
| **A. 표준안 (권장)** | 제외 | 2차 별도 | **24주 (6개월)** |
| B. 손든 추가 | 포함 | 2차 별도 | 27주 |
| C. 정산 포함 | 제외 | 포함 | 29주 |
| D. 풀스코프 | 포함 | 포함 | 32주 |
