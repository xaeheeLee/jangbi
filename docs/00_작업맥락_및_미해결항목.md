# 00. 작업 맥락 & 미해결 항목 (Claude 인수인계용)

> 이 파일은 **Claude Code 메모리에만 있던 맥락**을, 머신 이전(윈도우→맥) 후에도 잃지 않도록 docs로 옮겨 정리한 것입니다.
> 기존 계획문서(CLAUDE.md·dev_plan·기능명세)에 **아직 안 들어간 결정·미해결 항목·작업방식**이 핵심입니다.
> 새 환경에서 Claude는 이 파일을 읽고 맥락을 파악한 뒤, 필요하면 메모리에 다시 저장하세요.

## 1. 작업자 · 협업 방식

- **재희** — 회사 병행, 퇴근 후·주말에 **전중배(중장비 배차 매칭 앱)** 를 **1인 외주 개발**. Claude Code(Opus) 적극 활용.
- 고객(전중배 운영 사업자)과 **직접 미팅·계약**. 개발 외에 **계약·견적·PG·법무 등 비개발 실무**도 함께 도움 필요 → 답변 시 **개발자 관점 + 외주 사업자 관점** 둘 다 고려.
- **⚠️ 작업방식 선호 (중요):** 고객 미확정 항목이 섞인 작업에서는 **핵심 계획문서(계획서·HANDOFF·CLAUDE.md)를 곧바로 일괄 수정하지 말 것.** 대신 "확정된 것만 신규 문서화 + `[확인필요]` 표시"하고, 핵심 문서 대수술은 **고객 답변 확보 후 한 번에**. 진행 전 어느 쪽을 원하는지 재희님께 확인.

- 이미 docs에 반영 완료된 결정(참고만): ID 컨벤션(`jobs.job_no`=YYMMDD-NNNN, `member_no`=6자리 숫자), 지정배차(3건당 우선배차권·5분 윈도우), 허용 장비 다중(OR 매칭), 승인 SLA(영업일 3일), 별점(공개)≠평점(`admin_score`, 비공개 매칭 2순위)·사진인증(`job_photos`).

## 3. 결정됐지만 계획문서에 아직 미반영된 사항

- **🚫 뻥배차(가짜/유령 배차) 비구현·비문서화 확정.** 실제 기사 없이 활성도를 부풀리는 기능 → 돈 내는 기사 기만(표시광고법 허위·과장, 전자상거래법 기만, 충전·일일차감 유인 시 사기 소지). 콜드스타트 대안만 허용: **운영자 대리등록(실제 일감)·베타/샘플 라벨·지역집중 런칭·초기 인센티브(일일차감 면제 등)·정직한 집계.**
- **UX 상태 규칙 (계획문서 미반영, 목업엔 적용):**
  - **'우선 지원중'(라이브 카운트다운)** 은 우선배차 윈도우 동안 **일감 상세 화면에서만** 노출. 목록(배차·캘린더)에선 안정 상태 **'배차 대기'** 로 표기, 우선배차권은 작은 티켓 표식만.
  - **서류 '검토중'** 은 5종 일괄 심사 상태 → **승인 대기 화면에서만** 노출. 회원가입 업로드 단계의 '완료'는 '업로드 완료' 의미.
  - **색:** 네이비 기본 + **빨강은 우선배차/돈 빠짐 액센트로만** 절제 사용.
- 관리자 **칭찬 메시지 + 별점 가점(+1/+2)** 발송(일감매칭 화면) → 기사 알림. **가점이 공개 유효별점에 가산되는지 / 별도 칭찬배지인지 정책 미확정.**

## 4. ⚠️ 미해결 — 고객 확정 필요 (목업엔 임시값)

> 이 항목들은 **다음 고객 미팅에서 확정**해야 하며, 그 전엔 임의로 계획문서를 고치지 말 것(§1 작업방식).

1. **서류 5종 목업↔문서 불일치:** 기능명세서 v1.1 기준 5종 = ①건설기계 조종사면허 ②건설기계 등록증 ③보험가입 증명서 ④사업자등록증 ⑤본인 사진(상반신·3개월내). 그러나 목업(⑧/⑫/⑮)엔 **통장사본·차량등록증**이 포함 → **문서 기준으로 목업을 맞출지** 고객 확인 대기. (출금 계좌는 **1원 인증**(예금주명 자동 대조) 화면으로 별도 등록)
2. **소개비 방향** — 발주자 수령·지원자 지급 10%로 가정 중. 최종 확정 필요.
3. **충전 부가세 · PG수수료 440P 표기 방식** — 사용자 노출 문구·약관 표현.
4. **인출 최소금액 · 소요일.**
