# 출시 전 정리·배포 체크리스트 (전중배)

> 개발 중 누적된 dev 산출물 정리 + 프로덕션 배포 준비. 마지막 업데이트: 2026-06-26.

## 0. 핵심 원칙
**프로덕션은 현재 dev Supabase 프로젝트(ref `twrxjkivuxwgekgyvejv`)가 아니라 신규 프로젝트로 띄운다.** 그러면 아래 dev 시드는 자동 무력화(no-op)되고 테스트 데이터도 안 딸려간다.

## 1. dev 코드 — ✅ 제거 완료(이 커밋)
- `app/lib/main.dart` DEV_AUTOLOGIN_* 자동로그인 → 제거
- `app/lib/core/router/app_router.dart` PREVIEW_ROUTE → 제거
- `app/lib/features/home/home_shell.dart` PREVIEW_TAB → 제거
- (재검증 스크린샷 필요 시 임시 dart-define로 재도입 가능)

## 2. dev 시드 마이그레이션 — 신규 prod에선 자동 no-op
모두 `WHERE phone='010-9999-0001/8888/7777'` 등 **테스트 유저 존재 시에만** 동작하도록 가드됨. 신규 prod DB엔 해당 유저가 없어 무해. (이미 적용돼 history에 있으니 파일 삭제는 금지.)
- `090001_dev_seed` · `090003_dev_priority_job` · `090004_dev_today_job` · `090005_dev_worker_b` · `090006_dev_worker_c` · `100004_dev_low_balance`
- `100001_admin_foundation` 끝의 `UPDATE … is_admin=true WHERE phone='010-9999-0001'` → prod 무해(해당 유저 없음). **prod에선 실제 관리자 계정에 수동으로 `is_admin=true` 부여 필요.**

## 3. 현재 dev DB 테스트 데이터 (dev 프로젝트 계속 쓸 경우만 정리)
- 테스트 유저 3명: 검증기사(010-9999-0001)·기사B(010-8888-0001)·기사C(010-7777-0001) → Supabase 대시보드 **Authentication → Users** 에서 삭제(프로필·일감·거래는 FK CASCADE/잔여 정리).
- 샘플 일감(`[시드]` 설명) · point_transactions(`시드충전`) · 기사C suspended 상태 등.
- **권장: prod는 신규 프로젝트 → 이 정리 불필요.**
