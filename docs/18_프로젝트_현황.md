## 2. 구현·검증 완료 (실DB로 동작 확인)

- **인증/가입(P1):** 전화+비밀번호(합성이메일 방식). 가입→트리거가 profiles(pending)+회원번호 생성→RLS. 승인대기→관리자 승인→정회원.
- **일감(P2):** 등록(일반/지정·다중 허용장비)·목록(필터·우선배차/모집중/배차완료)·상세.
- **매칭(P3):** 일반매칭(apply_general 즉시), 우선배차(apply_with_priority+배차권 소진+finalize 5단계 선정), 지정배차. **소개비 10% 기사→발주자 정산** 실DB 검증(350,000→35,000 / 480,000→48,000).
- **포인트/인출(P4):** 포인트 원장(balance_after), 인출신청(request_withdraw, 본인계좌+관리자 승인 게이트), 일일차감 1000p+잔액부족 박탈.
- **관리자웹(P7):** 대시보드·회원승인·인출승인·프리미엄·평점·세무Export(CSV)·일감·서류마스킹·사진·시스템현황. 관리자=`is_admin`, 관리자 전용 RPC 게이트.
- **알림(P5 인앱):** 인앱 알림센터·읽음·딥링크, 알림 필터 설정.
- **FCM 푸시:** 클라이언트 토큰등록(device_tokens) + send-push Edge Function(서비스계정→FCM v1) + notifications INSERT 트리거. `{"sent":1}` + 디바이스 수신 확인. 안드로이드 검증, iOS는 Apple 계정 대기.
- **캘린더(P6):** 월 그리드·일감 점·선택일 카드.
- **자동화:** pg_cron 7스케줄(일일차감·윈도우 마감·만료·알림·정리) 활성·검증.
- **보안 수정:** profiles 자가수정 차단, apply RPC auth.uid() 강제, 관리자/cron 함수 권한 제한, RLS 무한재귀 해결.

## 3. 미완 / 남은 작업

### 3.1 기능 공백 (블로커성)
- **충전 실결제(토스 가상계좌):** UI·계산·confirm_charge RPC는 있으나 **PG 웹훅 미연동(스텁)**. → **PG 계약 후** 연동. (충전이 막혀 돈 흐름 종단 미완.)
- **인출 실지급:** 신청·승인 로직은 됨. 실제 송금은 운영(관리자가 승인 후 이체). **선불업 등록 전 실운영 불가**(규제).

### 3.2 placeholder / 후속 구현
- **지도:** 카카오맵 SDK 미연동(상세 화면 지도 블록 placeholder).
- **일감등록 사진 첨부**: 일부 스텁.
- **서류 마스킹 뷰어:** 관리자웹에 틀은 있음, 실제 마스킹 도구는 외부.
- **지역 마스터 데이터:** 상수 처리(지역 코드 마스터 테이블 없음).

### 3.3 출시 준비 (코드 외)
- 앱 아이콘·스플래시(브랜드 로고 적용), bundle id 확정, 버전·서명(Android keystore / iOS 인증서).
- 스토어 등록(Apple Developer ₩129,000/년, Google Play ₩35,000).
- 약관·개인정보처리방침(사업자등록·법무 후).

### 빌드·실행
```bash
# 모바일 앱 (app/)
flutter run --dart-define-from-file=dart_define.json          # 시뮬/에뮬/기기
flutter build apk --release --dart-define-from-file=dart_define.json   # 안드로이드 APK
# 관리자웹 (admin/)
npm run dev    # localhost:5173 (npm install 후)
```
