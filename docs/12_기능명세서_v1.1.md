## 1. 제품 개요

### 1.1 정의
포크레인(굴착기) 등 중장비 기사 간 **일감 등록·배차 매칭** 모바일 플랫폼. 단일 사용자 유형(중장비 기사)이 일감을 올리고(발주) 동시에 배차를 받을 수 있습니다.

### 1.2 비즈니스 모델 (전중배 = 기사 간 배차 중개)
- 발주자 A가 직접 수행 못 하는 일감을 등록 → 기사 B가 배차받아 수행
- B는 일감 금액의 **10%를 소개비로 A에게 포인트 지급** (현재 플랫폼 수취 0원)
- 일당 본금(예: 50만원)은 앱 외부에서 직접 수령 (앱은 "계산서 발행 권장" 안내만)
- 정상 운영 후 매칭 건당 **플랫폼 수수료 1000p(기사 부담)** 도입 예정

### 1.3 수익 구조

| 구분 | 내용 | 귀속 | 부담자 |
|------|------|------|--------|
| 구독료 | **하루 1000p 차감** (정회원 유지 조건) | 플랫폼 | 전 회원 |
| 충전 부가세 | 충전액의 **10%** | 플랫폼/세무 | 충전자 |
| 충전 PG 수수료 | 토스 가상계좌 **440p/건**(400원+VAT) | (토스 납부) | 충전자 |
| 일감 소개비 | 일감 금액의 **10%** (B→A) | 발주자 A | 배차받은 기사 B |
| 플랫폼 수수료 | 매칭 건당 **1000p** (현재 0, 추후 적용) | 플랫폼 | 배차받은 기사 B |

> 모든 수수료/부담금 금액은 **설정 테이블 상수**로 분리하여, 계약·정책 변경 시 코드 수정 없이 조정 가능하게 구현합니다.

### 1.4 기술 스택

| 구분 | 기술 |
|------|------|
| 모바일 | Flutter (iOS / Android) |
| 상태관리 | Riverpod / 라우팅 go_router |
| 백엔드·DB | Supabase (PostgreSQL + Realtime + Auth + Storage) |
| 결제 | 토스페이먼츠 가상계좌 (포인트 충전) |
| 지도 | 카카오맵 (일감 위치 표시용) |
| 푸시 | FCM |
| 스케줄러 | pg_cron |
| 오류 모니터링 | Sentry(또는 동급) — 원격 디버깅 대비 |
| 관리자 | **PC 웹 페이지 (별도 산출물)** |

> 거리 기반 매칭은 폐지되어 매칭에 GPS를 사용하지 않으며, 지도는 일감 위치 표시 용도로만 사용합니다.

## 2. 사용자 유형 및 권한

| 유형 | 설명 | 주요 권한 |
|------|------|----------|
| **정회원 기사** | 가입·승인 완료 + 포인트 잔액 ≥ 1000p | 일감 등록/지원, 매칭, 충전, 인출 |
| **준회원 기사** | 포인트 부족(<1000p)으로 정회원 박탈된 상태 | **전체 차단** — 충전만 가능 (열람·지원 불가) |
| **승인대기 기사** | 가입했으나 관리자 승인 전 | 앱 사용 차단(대기 화면) |
| **프리미엄 배차인** | 관리자가 명단에 등재한 기사 (정회원의 부가 속성) | 매칭 최우선 + **하루 1건 제한 면제** (비공개) |
| **관리자** | 운영자 (PC 웹) | 회원 승인, 공지, 별점 가감, 프리미엄 명단 관리, 회원/포인트/세무 관리 |

> - **회원번호:** 모든 회원에게 사람이 읽기 쉬운 고유 **회원번호**를 발급(지정배차·고객센터 식별용).
> - **프리미엄 배차인**은 별도 회원 유형이 아니라 정회원에 부여되는 플래그입니다. 사용자에게는 비공개입니다.

## 3. 모바일 앱 기능 명세

### 3.1 인증 / 회원가입

| 기능 | 상세 |
|------|------|
| 로그인 | **핸드폰번호(ID) + 비밀번호**. 세션 장기 유지(자동 풀림 없음), 로그아웃 시 종료 |
| 본인 인증 | **카카오 알림톡 OTP** [추후 구현 — 1차는 보류]. 카카오 비즈채널은 **우리(전중배)가 개설 후 고객(사업자)에게 이관** |
| 회원가입 단계 | ① 약관 동의 → ② 정보 입력 → ③ 서류 업로드 → ④ 신청 완료 → ⑤ 관리자 승인 대기 |
| 가입 정보 | 이름, 핸드폰번호, 비밀번호, 자격증 종류, 사업자 정보, **사업자 이메일·사업자 계좌번호** |
| 업로드 서류(5종) | ① 건설기계 조종사 면허증 ② 건설기계 등록증 ③ 보험가입 증명서 ④ 사업자 등록증 ⑤ 본인 사진(어깨 보이는 상반신, 3개월 내) |
| 회원번호 발급 | 승인 시 고유 회원번호 발급 |
| 승인 흐름 | 관리자(PC)가 서류 5종 검토 후 승인 — **보통 영업일 3일 이내**. 승인 전까지 전체 차단(승인 대기) 화면 |

> 가입 정보·서류는 고객이 추후 추가/변경 요청 가능(변경관리 절차 적용).

### 3.5 매칭 (핵심 로직)

**우선순위 (높은 순) — 내부 비공개 (2026-06-14 미팅 반영):**

1. **프리미엄 배차인** (관리자 명단, 여럿이면 먼저 지원한 사람)
2. **평점(관리자 점수) 높은 순** — 기본 50점, 관리자 ±1
3. **일감횟수(직전 3개월 발주이력) 많은 순**
4. **별점(발주자 평가) 높은 순**
5. **일반 배차 (선착순)**

> 2~4는 **우선배차권 보유(또는 프리미엄) 우선지원자**끼리의 정렬 순서이며, 30초 윈도우 종료 후 우선지원자가 없으면 일반 공개(선착순)로 전환됩니다.
> **평점(2순위):** 전원 기본 **50점**, 관리자가 **±1만** 조정. 사용자에겐 본인 50점만 보이고 **2순위로 쓰이는 사실은 비공개**(대부분 50점이라 의미 없는 지표로 인식). 변경은 사유·이력 기록.
> **일감횟수("발주 이력", 3순위):** 직전 3개 캘린더월 매칭 성사된 일반 발주 건수(지정배차 제외). 수치·작동 방식 비공개.
> **별점(4순위):** 발주자가 부여한 0~5점(베이지안 유효별점). 3.6 참조.

| 항목 | 상세 |
|------|------|
| 우선배차 윈도우 | 일감 등록 시 조건 맞는 우선배차권 보유자가 있으면 **N초** 윈도우 개시 (**기본 30초, 설정값** — 60/90초로 조정 가능) |
| 우선 지원 | 윈도우 내 우선배차권 보유자만 지원 가능. **프리미엄 배차인은 배차권 없이도 우선 지원 가능** |
| 채택 | 윈도우 종료 후 위 우선순위로 채택 |
| 프리미엄 처리 | 프리미엄 배차인 지원 시 별점/발주이력과 무관하게 **최우선 채택**. 단 **즉시 채택하지 않고 윈도우 종료 후 확정**(비공개 유지). 2명 이상이면 **먼저 지원한 사람** |
| 일반 배차 | 우선배차권자 미존재 시 또는 윈도우 내 우선지원자 없으면 일반 공개 → **선착순 즉시 매칭** |
| 1일 1건 | 기사는 **하루 1건만 배차** 가능. **기준 = 배차를 수락한 날(같은 날 2건 수락 불가)**. **프리미엄 배차인·지정배차는 제외** |
| 탈락자 | 우선배차권 반환(만료일 최소 보장) |
| 우선배차권 유효기간 | 발급 후 일정 기간 만료 — **기간은 조사 후 확정**(현재 30일 잠정·설정값) |
| 수익화(향후) | **우선배차권 유료 판매**(프리미엄과 무관) 검토 — **1년 후 plan**, 현재 제외 |
| 소개비/수수료 | 매칭 확정 시 기사 B → 발주자 A로 **소개비 10% 포인트 이전** + (추후) **플랫폼 수수료 1000p 차감**. 포인트 부족 시 지원 단계에서 차단 |

### 3.6 평점 / 별점 / 평가

| 항목 | 상세 |
|------|------|
| 별점(평가 주체) | 일감 수행 후 **발주자가 0~5점** 부여 (배민·당근식 리뷰). 매칭 **4순위** 지표 |
| 유효별점 | **베이지안 가중평균** — (평가수×본인평균 + m×전체평균C)/(평가수+m), 초기 C=3.5·m=5(설정값) |
| 신규 뱃지 | 첫 5회 일감 동안 "신규" 표시 |
| **평점(관리자)** | **별도 지표.** 전원 기본 **50점**, 관리자 **±1** 조정 → 매칭 **2순위**. 사용자엔 50점만 노출(우선순위 용도 비공개). 변경 사유·이력 기록 |

> 기존 '특별 별점(관리자 +1/+2 가산)'은 신설 **평점(2순위)**으로 **일원화**합니다. 별점은 순수 발주자 평가로 유지(관리자 가산 없음).

### 3.12 사용자 차단 (신규)

> **목적:** 함께 일하고 싶지 않은 상대를 서로 안 보이게 하여 불필요한 매칭/마찰 방지.

| 항목 | 상세 |
|------|------|
| 차단 등록 | 마이페이지 또는 상대 프로필/일감 상세에서 "차단" → 본인 **차단 목록**에 추가 |
| 차단 효과(목록) | A가 B를 차단하면 **A의 일감 목록·알림에서 B가 올린 일감이 보이지 않음** |
| 차단 효과(매칭) | 상호 매칭 방지 — **B는 A의 일감에 지원·매칭 불가**(반대 방향도 동일하게 가림 처리) |
| 차단 해제 | 차단 목록에서 제거 시 즉시 정상 노출 |
| 관리 | 마이페이지 "차단 목록" 화면에서 조회/해제 |

> 차단은 **단방향 의사(A→B)** 로 등록하되, 효과는 **양방향 비노출 + 매칭 차단**으로 적용하여 어색한 매칭을 원천 차단합니다.

- 기사/업체 회원 분리, 아이핀 인증

> **변경:** v1.0에서 범위 제외였던 **포인트 현금 인출**은 v1.1에서 **범위 포함**으로 변경되었습니다(전자금융거래법 검토 후 인출 범위 확정 필요).

> **현재 모든 정책 항목 확정.** R4(선불업)는 고객의 법적 절차이며, 개발 측은 인출 시 **본인 명의 계좌 한정·관리자 승인 게이트·거래 원장**을 구현하여 규제 대응을 보조합니다.

## 8. 가정 및 제약
