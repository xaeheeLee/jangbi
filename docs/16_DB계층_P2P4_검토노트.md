## 3. 자체 리뷰에서 잡아 수정한 것
- **인출 RPC 누락 → 추가**(request/approve/reject_withdraw, 본인계좌+관리자게이트+원장).
- **일일차감 후 잔액<1000 즉시 suspended**(CLAUDE.md 정의 일치).
- **배차권 환불 7일 하드코딩 → `app_settings.ticket_refund_min_days`**.
- **expire_old_jobs가 미완 윈도우를 매칭 없이 만료 → finalize 먼저** 호출.
- **알림 중복(만료·잔액경고) → 당일 1회 가드**.
- **장비매칭 NULL 전파 차단**(미등록 기사 부당 제외 방지).
- 에러코드 `JOB_UNAVAILABLE` 통일.

리뷰 정상 확인(무수정): 락 순서, SECURITY DEFINER+search_path, 매칭 ORDER BY(별점100% 베이지안), settle_referral 원장, 서류 마스킹, 거리/GPS 미사용.

## 4. ⚠️ 당신 결정이 필요한 항목 (아침에 확인)
1. **일정 충돌 판정 입도** — `jobs.work_date`가 단일 timestamp라 "같은 날 다른 시각" 중복은 못 막음. 하루1건(수락일 기준)이 일반회원은 커버하나 **프리미엄·지정배차는 면제**라 같은 날 2건 가능. 시간구간(work_end/duration) 추가할지 결정 필요. (현재: 동일 시각만 충돌 처리)
2. **인출 거절 환불 원장 타입** — 전용 타입이 enum에 없어 `admin_adjust + memo='인출 거절 환불'`로 처리. 세무 분류상 전용 타입 원하면 신규 마이그레이션으로 enum 확장.
3. **VAT 추적** — 충전 VAT는 `charges.vat` 컬럼에만 있고 원장(point_transactions)엔 없음(포인트는 원금만 발급 = 정상). 세무 export는 charges JOIN으로. 이 방식 OK인지 확인.
4. **관리자 게이트** — `admin_adjust_score`·`approve/reject_withdraw`는 함수 내부에 관리자 검증이 없음(관리자웹 service_role 래퍼 전제). P7 관리자웹에서 래퍼 필요.
5. **사진 1년 보관 후 삭제(cron⑦)** — 파괴적 삭제는 SQL 범위 밖으로 두고 건수 집계만. 실제 Storage 삭제·백업은 관리자 Edge 배치로.
