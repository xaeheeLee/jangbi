# 고객 추가 요구사항
* 일감 매칭 이후에 발주자는 기사의 정보(회원가입시 받은 5가지 정보) 확인 가능(캡쳐는 불가)
	* 배차인의 신뢰를 받는 장치
* (비밀사안) 3개월 기준으로 '올린 일감 갯수'를 카운팅(하기 '지정배차'는 제외)
	* 이 '올린 일감 갯수'는 일을 자주 올리는 분들을 위한 장치. 내부 비밀 사안으로 기존 우선 배차권 경쟁시 우선순위로 됨. 하기 우선순위로 변경.
		* 우선순위
			* 1. '프리미엄 배차권'
			  2. 우선배차권 + '올린 일감 갯수' (동일 일감 개수면 별점)
			 3. 우선배차권 + 별점
			 4. 일반배차(선착순)
* 지정배차
	* 본래 발주자와 기사가 알고 있는 사이에서 굳이 어플 없이 일을 진행하던 사람들을 어플 내로 흡수하기 위한 장치
	* 지정배차는 해당 일감 비밀번호 or 회원번호를 입력하여 해당 인물만 일감을 받을 수 있음.
	* 다른 user 도 해당 일감을 볼 수 있게 함
		* 거짓 일감 모니터링은 기본적으로 관리자도 하지만, 다른 유저도 같이 할 수 있도록, 또한 일감이 많아 보이는 효과.
	* 굳이 어플을 사용하여 귀찮음, 수수료를 내지만 3회당 우선배차권 1회를 부여하도록
* 필터링
	* 필터링이 꼭 구현 되도록 확인요청.
		* 일감목록 - 일감 리스트 내에서 본인의 지역/장비 를 선택하여 해당 일감을 볼 수 있도록.
		* 알람 - 모든 일감의 알람이 울리는 것이 아닌 지역/장비 를 선택한 일감들만 알람이 울릴 수 있도록.
* 서버관련
	* 서버가 터지는 문제가 없도록 구현 부탁함.
	* 특히나 유지 보수시에도 1인개발이기 때문에 버그 발생 시, error를 claude, mac 을 활용하여 회사에서 원격으로 디버깅 할 수 있도록 구현해야할듯.
* 일감 발주 관련
	* 모든 일감은 '우선배차권'을 활용하여 일감을 올리는 시스템임을 확인
* PC 관리자 페이지
	* 프리미엄 배차는 관리자가 프리미엄 배차인 리스트를 관리하여 관리자가 직접 넣고 빼고 할 수 있음.
	* 회원 관리
		* 회원별로 어떤 일감을 수행하였고 어떤 일감을 올렸는지 db 관리 필요
		* 포인트 또한 충전/사용을 관리해야함
		* 이를 통하여 세무 관련 일을 할 수 있도록 관련된 db 저장 필요
	* 관리센터(운영실)
		* 카톡 채널 + 고객센터 번호로 운영 될 예정.
		* 카톡 채널은 로그인시 사용 및 문의사항등을 받는 창구로 운영 예정.
