# 장비나라 — 2차 미팅 결정 체크시트 (v1.1)

## 🟦 우선순위 1 — 매칭 모델 (가장 중요)

### Q1. 손든 사람 기능, 넣을까요?
발주자가 손든 기사 리스트를 보고 **직접 채택**할 수 있는 기능입니다.
- ☐ A. **제외** (자동 매칭만, 기존 합의안) — 공정성 우선, 일정 변동 없음
- ☐ B. **포함** — 발주자 재량 ↑, 단 **+3주** · 우선배차권 환불 정책 필요

> 개발측 의견: 우선배차권을 받은 기사가 손든 기사에 밀리면 배차권 가치가 사라집니다. "공정 배차"가 핵심 가치라면 A안 추천.

### Q2. 매칭 점수, 어떻게 계산할까요?
- ☐ A. **별점 70% + 거리 30%** (기존 합의안)
- ☐ B. **고객 예시안** — 콜 수행 40% + 정시 배차 30% + 회원 평가 50% − 취소 패널티 20% (거리 미반영)
- ☐ C. **거리 반영 여부만 정하고, 비율은 개발측 초안 보고 결정**

> **포크레인 작업이 같은 시/도 안에서 도나요, 멀리 광역으로 이동하나요?** 
> → 거리 반영 비중이 여기서 갈립니다.

### Q3. 우선배차 시간 (참고 — 30초로 합의됨)
변경 없으시면 **30초로 진행**합니다. 다만 푸시 알림은 휴대폰 상태에 따라 늦게 도착할 수 있어 참고로만 공유드려요.

| 상태 | 30초 중 실제 가용 시간 |
|------|---------------------|
| iPhone (정상) | 약 25~27초 ✅ |
| 갤럭시 (정상) | 약 18~22초 ⚠️ |
| 갤럭시 (절전모드) | 0~15초 ❌ |

- ☐ 30초 유지 / ☐ 60·90초로 늘림 / ☐ 운영하며 조정
> 앱을 켜두면 0.1~0.3초 만에 바로 뜹니다(푸시와 무관). 보완책: 고우선순위 푸시 / 첫 실행 시 절전모드 안내 / 카카오 알림톡 병행(별도 비용).

## 🟦 우선순위 2 — 회원 정보 및 가입 정책

### Q4. 회원가입 시 받을 정보
이름·휴대폰·비밀번호·약관·자격증 사진·장비 정보는 **이미 확정**입니다. 아래만 결정해 주세요:

| 항목 | 받을지 | 개발측 의견 |
|---|---|---|
| 생년월일 | ☐ 받음 / ☐ 안 받음 | 받음 권장 (연령 통계·본인 확인) |
| 성별 | ☐ 받음 / ☐ 안 받음 | 선택 |
| 이메일 | ☐ 필수 / ☐ 선택 / ☐ 안 받음 | 선택 권장 (알림·계정 복구) |
| 자격증 종류 명시 | ☐ 명시 / ☐ 안 함 | 명시 권장 (중장비조종사면허 등) |
| 사업자 정보 | ☐ 받음 / ☐ 안 받음 | 받음 권장 (세금계산서 대비) |
| 활동 지역 | ☐ 받음 / ☐ 안 받음 | 받음 권장 (매칭 정확도) |
| 차량번호 | ☐ 받음 / ☐ 안 받음 | 선택 |
| 경력 사항 | ☐ 받음 / ☐ 안 받음 | 선택 |
| 계좌번호 | ☐ 가입 시 / ☐ 매칭 후 공개 | 매칭 후 공개 권장 |

### Q6. 정회원 등급, 어떻게 줄까요?
- ☐ A. **가입(자격증 승인) = 자동 정회원** (개발측 권장, 단순)
- ☐ B. 별도 신청 + 관리자 추가 승인
- ☐ C. 활동지수 N점 이상 자동 승급
> 등급을 나눈다면: **정회원과 일반회원의 "기능 차이"가 무엇인지** 정해 주세요 
   (예: 우선배차권 사용 권한).

### Q7. 활동지수, 어떤 행동에 몇 점 줄까요?
일감 등록 / 손들기 / 매칭 성공 / 작업 완료 / 취소 페널티 등.
- ☐ 개발측 초안대로 시작 후 운영하며 조정 (권장)
- ☐ 직접 점수 지정 (메모: __________________________)
> 초안 예시: 일감등록 +5 / 매칭성공 +10 / 작업완료 +15 / 손들기 +2 / 취소 −20 / 상한 100점

## 🟦 우선순위 3 — 발주 화면 세부

### Q8. "1군 서류"란?
고객 자료(발주 화면)에 **"작업 총수 / 1군 서류"** 입력란이 있는데, 의미를 정확히 맞추고 싶습니다.
- 1군 서류가 무엇을 뜻하나요?
- 입력 방식: ☐ 드롭다운 선택 / ☐ 텍스트 입력 / ☐ 파일 업로드
- ☐ 필수 / ☐ 선택

### Q9. 이용 금액
- ☐ A. **사전 가격 제시** — 발주자가 가격을 정하고, 기사는 지원/거부 (가장 단순, 권장)
- ☐ B. **견적 요청** — 발주자 희망가 표시, 기사가 다른 가격 제시 가능 (협상=채팅 필요)
- ☐ C. **혼합** — 우선배차는 정가, 일반 매칭은 협상

### Q10. 작업 종류 태그
현재안: **가위 / 고무궤도 / 그라플 / 다짐기 / 돌집게 / 뿌레카 / 지게발 / 코아(천공) / 기타**

## 🟦 우선순위 4 — 향후 확장

### Q11. 작업일지·정산·거래명세서, 언제 넣을까요?
사업 락인 도구로 보이지만 1차에 넣으면 일정이 길어집니다.
- ☐ A. **1차에 포함** (총 29주)
- ☐ B. **출시 후 바로 2차 개발** (개발측 권장)
- ☐ C. **운영 안정화 후 추후 결정**

### Q12. 평점/평가 시스템, 어떻게 운영할까요?
- 평가 시점: ☐ 작업 완료 후 N일 이내 / ☐ 기타: ________
- 평가 항목: ☐ 별점만 / ☐ 별점 + 코멘트
- 미작성 시: ☐ 만료(권장) / ☐ 자동 5점 / ☐ 페널티
- 양방향 평가(발주자 ↔ 작업자): ☐ 진행 / ☐ 한쪽만
> 평가 시스템 자체를 2차로 미루고 1차는 데이터 수집만 할 수도 있습니다.

## 💰 견적 (4가지 시나리오)

| 시나리오 | 손든 사람 | 작업일지·정산 | 개발 기간 |
|---------|:---:|:---:|:---:|
| **A. 표준안 (권장)** | 제외 | 2차 별도 | **24주 (6개월)** |
| B. 손든 추가 | 포함 | 2차 별도 | 27주 |
| C. 정산 포함 | 제외 | 포함 | 29주 |
| D. 풀스코프 | 포함 | 포함 | 32주 |

> **권장 흐름:** A안 1차 출시 → 4~6주 운영 안정화 → 2차 모듈 추가 (총 35주, 약 8~9개월)

**일정(A안):** 4주 회원가입 → 8주 일감등록 → 13주 매칭 → 16주 푸시 → 20주 캘린더 → **24주 1차 출시 🎉**

**운영비(월):** 초기 ~300명 약 3.7~4만 원 / 1,000명 약 4~4.5만 원 / 3,000명 약 4.5~5만 원
**1회성:** Apple Developer 연 12.9만 원 / Google Play 1회 3.5만 원

### 월 운영비(예상)

| 항목 | 비용 | 비고 |
|------|------|------|
| Supabase Pro (서버·DB) | 약 3.5만 원 | 8GB DB, 100K MAU, 자동 백업 (고정비) |
| SMS 인증 (알리고 등) | 약 0.4~1.2만 원 | 건당 약 20원 · 월 가입자 최대 500명 가정 |
| 카카오맵 JS API · 로컬 API (주소→좌표) | **0원** | 지도 표시 무제한 무료 · 주소 변환 일 10만 건 무료 |
| Firebase FCM 푸시 | **0원** | 무제한 무료 |

> 즉, **사실상 고정비는 Supabase + 문자 인증뿐**이고 지도·푸시는 무료입니다.

> 사용자가 **3,000명 규모로 성장해도 무료 한도 안에서 매우 안전**합니다.
