# 장비나라 (포크레인 배차 매칭 앱) — 프로젝트 핸드오프 문서

> **이 문서의 목적**
> 웹 Claude와의 작업을 로컬 환경(Claude Code)으로 이어받기 위한 종합 컨텍스트 문서입니다.
> 로컬 Claude가 이 문서를 읽고 이전 맥락을 모두 이해한 상태에서 작업을 시작할 수 있도록 작성되었습니다.

---

## 📌 즉시 알아야 할 핵심 정보

### 프로젝트 개요

- **앱 이름:** 장비나라 (정식 명칭, 이전 가칭 "전중배")
- **앱 목적:** 포크레인 기사들 간 일감 등록 및 배차 매칭 모바일 플랫폼
- **고객:** 전중배 (장비나라 운영 예정)
- **개발자:** 1인 개발 (재희, 회사 다니며 퇴근 후 + 주말)
- **현재 단계:** 1차 미팅 완료, 2차 미팅 자료 준비 끝, **개발 착수 직전**

### 기술 스택 (확정)

| 구분 | 기술 |
|------|------|
| Frontend | Flutter (iOS / Android, 폴더블폰까지 대응) |
| 상태 관리 | Riverpod |
| 백엔드 / DB | Supabase (PostgreSQL + PostGIS + Realtime + Auth + Storage) |
| 지도 | 카카오맵 JavaScript API (WebView 통합) + 카카오 로컬 REST API (주소 검색·Geocoding) |
| 푸시 알림 | FCM + Supabase Edge Functions |
| 스케줄러 | pg_cron (Supabase 내장) |
| 개발 도구 | Claude Code (Opus), MacBook Air M4 |

### 핵심 일정

- **총 개발 기간:** 24주 (약 6개월) — 1인 퇴근 후 개발 기준
- **변동 범위:** ± 4주
- **변동 요인:** 회사 업무 영향, 외부 API 디버깅, 실기기 테스트

## 🎯 최근 결정사항 요약 (2차 미팅 준비 단계)

### 1차 묶음 — 발주 화면

| 항목 | 결정 |
|------|------|
| 장비 종류 | 포크레인만 1차 개발, DB는 확장 가능 구조 |
| 발주자/작업자 분리 | "발주자 = 기사 본인" 가정으로 1차 진행 |
| 1군 서류 | 고객 협의 필요 (미팅 안건 Q8) |
| 작업 시간 범위 + 사용시간 칩 | **적용** (P1-2를 P0로 격상) |
| 작업 종류 태그 다중 선택 | **적용** (가위/고무궤도/그라플/다짐기/돌집게/뿌레카/지게발/코아(천공)/기타) |
| 이용 금액 입력 | **적용** (모델은 고객 협의) |
| 임시저장 / 사진 첨부 / 3단계 스텝퍼 | **적용** |

### 2차 묶음 — 매칭 흐름

| 항목 | 결정 |
|------|------|
| 손든 사람 기능 | 별도 견적 (포함 시 +3주, 약 13일 작업) |
| 매칭 점수 거리 반영 | 고객 협의 필요 |
| 우선배차 시간 | **30초** 합의 (단, 푸시 도달 고려 미팅에서 재확인 필요) |
| 캘린더 4단계 색상 | **적용** |
| 일감 완료 흐림 처리 | **적용** |
| 폴더블폰 대응 | **포함** (1주 추가, 반응형) |
| PC/태블릿 대시보드 | **1차 제외** |

### 3차 묶음 — 회원가입 / 우선배차 / 평점

| 항목 | 결정 |
|------|------|
| 회원가입 흐름 | 통합안 5단계 (약관 → 정보 → 인증 → 자격증 → 완료) |
| 기사/업체 회원 분리 | **1차 제외** (기사만) |
| 아이핀 인증 | **1차 제외** (휴대폰 인증만) |
| 비밀번호 인증 추가 | **포함** (운영비 절감) |
| 자격증/장비 보유증 업로드 + 관리자 승인 | **포함** |
| 관리자 모드 | **앱 내 구현** (별도 웹 X) |
| 우선배차권 발급 | **일감 등록 시** 등록자에게 1장 |
| 우선배차권 만료 | **발급 후 30일** |
| 활동지수 / 정회원 등급 | 고객 협의 필요 |

### 4차 묶음 — 향후 확장 모듈

| 항목 | 결정 |
|------|------|
| 커뮤니티 채팅 (그룹/지역방) | **1차 제외, 향후에도 미정** |
| 작업일지 + 정산 + 거래명세서 | **1차 제외, 추가 모듈로 별도 견적** (+5주, 약 22일) |
| 신고 시스템 | **제외** |
| 100명 락인 전략 | 고객 사업 전략 (우리 작업 영향 없음) |

## 🚀 로컬에서 작업 시작 가이드

### Step 1: 환경 셋업

```bash
# Flutter 설치 확인
flutter --version
# 3.x 이상 권장

# Supabase CLI 설치
brew install supabase/tap/supabase

# 프로젝트 폴더 생성
mkdir jangbi_nara
cd jangbi_nara

# Flutter 프로젝트 생성
flutter create app --org com.jangbinara --platforms ios,android
cd app
```

### Step 2: 핵심 패키지 추가

```yaml
# pubspec.yaml
dependencies:
```

1. https://supabase.com 에서 프로젝트 생성
