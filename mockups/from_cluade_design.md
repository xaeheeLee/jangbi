# 핸드오프: 전중배 — B 카드형 디자인 컨셉

> Claude Code(또는 개발자)에게 전달하는 **디자인 정본 문서**입니다.
> 이 문서 하나 + `전중배 시안 - B 카드형.html` 만으로 앱 전체 화면을 일관된 비주얼로 구현할 수 있도록 작성했습니다.

## 0. 이 번들을 어떻게 쓰면 되나

- 함께 들어있는 **HTML 파일은 "디자인 레퍼런스"**입니다. 그대로 복사해 출시하는 코드가 아니라, **의도한 외형과 동작을 보여주는 목업**이에요.
- 할 일은 이 HTML 디자인을 **타깃 코드베이스의 환경(React / React Native / Flutter / SwiftUI 등)에 맞춰 다시 구현**하는 것입니다. 이미 정해둔 프레임워크/디자인 시스템이 있으면 그 패턴을 쓰고, 없으면 프로젝트에 가장 적합한 스택을 골라 구현하세요.
- **HTML은 단 3개 화면(로그인 · 일감 목록 · 일감 상세)만** 들어 있습니다. 나머지 화면(배차·캘린더·지갑·MY·회원가입·장비등록 등)은 아래 **§7 "새 화면에 적용하는 법"**의 규칙을 따라 **이 디자인 언어로 일관되게 새로 그려서** 구현하면 됩니다.

## Fidelity: High-fidelity (하이파이)

색·타이포·간격·라운드·그림자까지 **최종 확정값**입니다. 픽셀 단위로 충실히 재현하세요. 단, 폰 베젤·펀치홀·제스처바는 **목업용 기기 프레임**일 뿐 앱 UI가 아니므로 구현 대상이 아닙니다.

## 1. 디자인 컨셉 한 줄 요약

**"일감마다 떠 있는 라운드 카드 + 넉넉한 여백."**
흰 배경 위에 그림자로 떠 있는 카드, 신뢰감 있는 **네이비**를 주색으로, **빨강**은 '우선배차/긴급/주행동'에만 절제해서 사용. 정보는 카드 안에 위→아래로 컴팩트하게 쌓고, 시간에 민감한 정보(우선배차 마감)는 **실시간 카운트다운**으로 강조.

원칙:
- **네이비 = 브랜드/신뢰/기본 행동**, **빨강 = 긴급/우선/돈 빠지는 것**, **파랑계열 = 보조 정보 태그**.
- 카드는 항상 흰색 + 라운드 18px + 옅은 그림자. 강조 카드만 빨강 테두리.
- 여백은 넉넉하되 카드 내부는 컴팩트(요소 간 7~10px).
- 한국어 UI. 숫자는 `tabular-nums`(고정폭)로.

## 4. 공통 컴포넌트 스펙

> 모든 화면은 아래 컴포넌트의 조합입니다. 새 화면도 이 부품으로 조립하세요.

### AppBar (상단바)
- 높이 54, 좌우 패딩 16. 좌측: 뒤로가기(`i-back`) 또는 로고+타이틀. 우측: 액션 아이콘 버튼(38×38), 알림은 우상단 빨강 점.
- 타이틀 16/800. 배경 흰색, 하단에 `1px line` 구분선(목록/상세).

### TabBar (하단 탭, 5개)
- 높이 64, 흰 배경, 상단 `1px line`. 탭: 아이콘 22 + 라벨 10/700.
- 비활성 `ink-3`, **활성 `navy`** + 아이콘 위 작은 네이비 인디케이터 바(22×3, 하단 라운드).
- 탭 구성: **일감 · 배차 · 캘린더 · 지갑 · MY**.

### Job Card (일감 카드) — 핵심 부품
구조(위→아래):
1. **상단 행** (`space-between`): 날짜 `12/700 ink-3` ↔ 상태 **Pill**
2. **위치 제목** `17/800`, 위 7px
3. **장비 태그들** (가로 6px 간격): `Tag` 칩들, 위 7px
4. **구분선 행** (`space-between`, 위 10px·border-top 10px): 스펙 `<b>궤도형</b> · 대형` ↔ 가격 `19/800 navy` + `<small>원</small>`
5. (우선배차만) **카운트다운** (바 형태, 위 10px)

변형:
- **우선배차 카드:** 테두리 `1.6px red` + 빨강 글로우 그림자.
- **일반 카드:** 테두리 `1px line` + `shadow-sm`.
- **배차완료 카드:** 내용 `opacity .5` + 중앙에 흰 알약 "배차완료" 잠금 라벨(체크 아이콘).

### Pill (상태칩) — 11/800, padding 4×9, radius 7
- `prio` 우선배차: 흰글씨 / `red` 배경
- `open` 모집중: `blue-ink` / `#E4EDFF`
- `done` 배차완료: `#64748B` / `#EAEDF2`

### Tag (장비 태그) — 11.5/700, `blue-ink` on `#EAF1FE`, radius 7. 회색 변형 `gray`(완료건).

### Model Badge (장비 기종 배지)
- 제목 우측에 우측정렬로 붙는 **2줄 세로 배지**: 위 작은 라벨 `기종`(10/800, blue-ink 60%) + 아래 코드 `04LC`(17/800, navy, 자간 .05em, tabular-nums).
- 배경 `#EAF1FE`, radius 10, 패딩 5×12, **min-width 62px**(코드 3~5글자 대응). `flex:none`. 제목은 `flex:1; min-width:0; nowrap; ellipsis`로 한 줄 유지.

### Countdown (우선배차 마감) — 2가지 변형
- **바(bar) 형:** 라벨 `우선배차 마감까지`(11.5/800 orange) ↔ 남은초 `24초`(orange, tabular). 아래 6px 트랙(`#FBE3D6`) + `linear-gradient(90deg, orange, red)` 채움 바. 목록 카드에서 사용.
- **링(ring) 형:** 38px 원형, `conic-gradient(red N%, #FBD5D5 0)`로 잔여비율 표시, 가운데 숫자(13/800 red). 옆에 `우선배차 마감까지` + `배차권 보유자 우선 지원 가능`. 상세 화면에서 사용.
- **동작:** 30초에서 1초까지 1초마다 감소, 0 도달 시 30으로 루프(목업용). 비율 = `남은초/30 × 100%`.

### Buttons — 높이 50~52, radius 14, weight 800
- `btn-navy`(주행동, navy + shadow-lift) / `btn-red`(우선/긴급, red) / `btn-white`(네이비 위 흰버튼) / `btn-ghost`(흰 배경 + 1.5px 보더, 보조 행동). `:active`에서 `scale(.985)`.

### Input — 높이 50, radius 13, 1.5px line. `filled` 변형: 배경 `#F4F6F9`, 보더 투명. 라벨 12.5/700 ink-2.

### Filter Chips — 높이 34, radius 999. `sel`(navy 채움) / `out`(흰+보더) / `add`(점선 보더, "+필터").

### FAB — 56×56, radius 19, `i-plus`. 네이비 또는 빨강 그라데이션, 우하단(탭바 위 80px).

### ListHead (목록 상단 카운트) — `전체 일감` `12건`(navy) … 우측 메타 `서울 · 오늘`. 전체 일감 개수 노출용.

### Map block (상세 지도) — 목업 지도(블록+도로+핀+"카카오맵" 칩). **실제 구현 시 지도 SDK(카카오맵 등)로 대체.**

### Logo Badge (로그인 로고) — 흰 라운드 정사각(132px, radius 34) + 그림자. 안에 브랜드 로고 이미지. 아래 워드마크 `전국중장비배차연합`(26/800).

## 5. HTML에 포함된 3개 레퍼런스 화면

### ① 로그인
- **네이비 히어로**(상단 그라데이션) + 아래에서 올라오는 **흰 시트**.
- 상단 중앙: 큰 로고 배지 → 워드마크 `전국중장비배차연합` → 서브카피 `믿을 수 있는 기사와 일감을 한 번에.`
- 시트: 휴대폰번호/비밀번호 입력(filled) → `자동 로그인` 체크 + `비밀번호 찾기` → **로그인 버튼(btn-navy)** → `회원가입` 링크 → 하단 3종 신뢰배지(자격 검증·빠른 배차·안전 거래).

### ② 일감 목록
- AppBar(로고+`일감`, 알림) → **ListHead `전체 일감 12건`** → 필터 칩 → 일감 카드 리스트(우선배차/모집중/배차완료 3종 상태 예시) → FAB(등록) → TabBar.
- 우선배차 카드에 **바 카운트다운** 실시간 작동.

### ③ 일감 상세
- AppBar(뒤로/`일감 상세`) → 지도 → 흰 시트: `우선배차 진행 중` pill → **제목 + 우측 기종 배지(`기종`/`04LC`)** → 주소·일시 → **금액/소개비(10%) 차감** 행 → **링 카운트다운** 박스 → 작업종류·장비조건 라벨/값 → 하단 액션 2버튼(`일반 지원`(ghost) / `우선 지원`(red)).

## 6. 인터랙션 & 동작

- **카운트다운:** §4 Countdown 참고. 실제 구현은 서버의 마감 시각 기준으로 계산(여기선 30초 루프 목업).
- **네비게이션:** 일감 카드 탭 → 일감 상세. FAB → 일감 등록. 하단 탭 5개 전환. 뒤로가기 → 이전.
- **버튼 press:** `scale(.985)` 살짝 눌림.
- **상태별 카드:** 모집중/우선배차/배차완료에 따라 Pill·테두리·딤 처리 분기(§4 Job Card).
- **목록 head 카운트:** 필터 결과 총 건수 반영.

## 7. 새 화면에 적용하는 법 (배차·캘린더·지갑·MY·회원가입·장비등록 등)

HTML에 없는 화면은 아래 규칙으로 **같은 디자인 언어**를 유지해 새로 구성하세요.

1. **레이아웃 골격:** `AppBar`(또는 네이비 히어로) → 본문 스크롤 → `TabBar`. 본문 배경은 `bg(#F7F8FA)`, 콘텐츠는 흰 카드로.
2. **모든 정보 묶음 = 카드.** 라운드 18 / 흰배경 / `shadow-sm` / 좌우 16 여백 / 카드 사이 10.
3. **색 규칙 고수:** 기본 행동·강조 숫자=네이비, 긴급/우선/돈 빠짐=빨강, 보조 정보 태그=연블루. 빨강 남발 금지.
4. **타이포 스케일(§2.2)와 간격(§2.5)** 그대로. 숫자는 tabular-nums.
5. **라벨/값 행**은 상세 화면의 `작업 종류 … 값` 패턴 재사용(좌측 라벨 ink-3, 우측 값 ink, 행 사이 1px line-2).
6. **주행동 버튼은 화면당 1개(navy 또는 red)**, 보조는 ghost.
7. **상태 표현은 Pill**, 수치/뱃지는 연블루 배지(기종 배지 패턴 재사용).
8. **빈 상태/로딩/에러**: 빈 상태는 중앙 정렬 아이콘+안내문(ink-2)+주행동 버튼. 스켈레톤은 `line`색 블록.
9. 화면 단위 라벨에 `data-screen-label`을 달아두면 리뷰/QA에 도움.

예시 매핑:
- **배차 탭** → 내 지원/배차 현황을 상태 Pill 카드 리스트로.
- **캘린더** → 월 그리드 + 날짜별 일감 점/카드.
- **지갑** → 상단 잔액/포인트 카드(네이비), 아래 정산·소개비 내역 라벨/값 리스트.
- **MY** → 프로필 카드 + 메뉴 리스트(자격 검증·내 장비·설정), 자격배지 신뢰요소.
- **회원가입/장비등록** → 로그인의 입력 필드·버튼 스타일 그대로, 단계형 폼.

## 9. 에셋

- **브랜드 로고:** HTML에 base64 PNG로 인라인되어 있습니다(로그인 큰 로고, 목록 앱바 소형 로고). 코드베이스에는 **원본 벡터/PNG 로고 파일**을 따로 받아 쓰는 걸 권장(고해상도). 본 목업의 base64는 임시 참조용.
- **아이콘:** 인라인 SVG `<symbol>`(§3). 라이브러리로 대체 가능.
- **폰트:** Pretendard (CDN: `cdn.jsdelivr.net/gh/orioncactus/pretendard`). 코드베이스 정책에 맞게 self-host 권장.
- **지도:** 목업은 가짜. 실제는 지도 SDK 연동.
